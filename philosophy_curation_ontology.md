# 철학 도서 큐레이션 MVP 온톨로지 최종안 v2.1

## 1. 문서 목적

이 문서는 한국어 철학 도서를 대상으로 한 설명형 추천 MVP의  
개념 구조, 추천 구조, 주제 체계, UX 원칙을 정의합니다.

이 MVP의 1차 목적은 판매가 아니라 아래 3가지를 검증하는 것입니다.

- 주제 선택
- 추천 책 상세 보기
- 재방문

---

## 2. 서비스 정의

이 서비스는 사용자가 주제를 선택하거나 자유롭게 질문을 입력했을 때,  
공개 메타데이터를 바탕으로 철학 도서를 탐색하고,  
철학 교수이자 철학 서점 운영자라는 추천자 페르소나의 관점으로  
설명형 추천 결과를 제공하는 시스템입니다.

이 서비스는 다음을 목표로 합니다.

- 철학적 맥락을 짧고 명확하게 설명
- 입문 난이도를 함께 제시
- 질문에 맞는 이유를 설명
- 과장 없이 신뢰 가능한 추천 경험 제공
- 핸드폰, 태블릿, PC에서 모두 깔끔한 선택 경험 제공

---

## 3. 서비스 범위

### 3.1 도서 범위
- 철학 도서만
- 한국어 / 국내 도서 중심

### 3.2 데이터 원칙
- 국립중앙도서관 오픈 API를 기본 외부 데이터 소스로 사용한다
- 외부 원문 전체나 대규모 외부 DB를 자체 축적하지 않는다
- 서비스 운영에 필요한 최소 구조만 내부 저장한다

### 3.3 추천 방식
- 주제 선택형 큐레이션
- 자유 입력형 큐레이션
- 자유 입력은 AI가 즉석 해석하되, 내부적으로 주제와 의미 태그에 연결한다

### 3.4 운영 원칙
- 초기에는 주제 20개를 모두 첫 화면에 노출한다
- 더보기는 두지 않는다
- 초기에 안 눌린다고 바로 버리지는 않는다
- 주제 선택 KPI를 누적 관찰한 뒤 조정한다

---

## 4. 추천자 페르소나

### 4.1 정의
추천자는 대학교 철학 교수이자 철학 서점을 운영하는 사람입니다.

서점을 찾는 손님에게 철학적 맥락과 입문 난이도를 함께 설명하고 싶지만 시간이 부족합니다.  
서점에는 아르바이트 직원이 있으나, 철학적 배경지식이 깊지 않아 책 설명을 충분히 하지 못합니다.  
이 AI는 그 부족한 설명을 대신하는 역할을 합니다.

### 4.2 역할
- 책의 철학적 맥락을 설명한다
- 입문 난이도를 설명한다
- 사용자의 질문과 책의 연결 이유를 설명한다
- 과장 없이 책 선택을 돕는다
- 책별/주제별로 커스텀된 설명을 제공한다

### 4.3 설명 원칙
- 짧고 명확하게 설명한다
- 철학적 맥락을 먼저 제시한다
- 입문자가 이해할 수 있는 언어를 사용한다
- 아는 척하지 않는다
- 공개된 책 정보와 일반적으로 알려진 철학적 맥락 안에서만 판단한다

### 4.4 금지 규칙
- 책 본문을 모두 읽은 것처럼 단정하지 않는다
- 장별 논증을 상세히 설명하지 않는다
- 확인되지 않은 인용을 생성하지 않는다
- 학계 평가, 영향력, 판매량을 근거 없이 단정하지 않는다
- 과장된 추천 문구를 사용하지 않는다

## 5. 온톨로지 핵심 구조

이 MVP는 아래 5개 핵심 자산과 1개 생성 결과 객체로 구성됩니다.

### 5.1 핵심 자산
1. Book
2. Theme
3. ConceptTag
4. CuratorPersona
5. InteractionLog

### 5.2 생성 결과 객체
6. CurationOutput

> CurationOutput은 핵심 도메인 엔터티라기보다  
> 위 자산을 바탕으로 생성되는 출력 결과 객체입니다.

---

## 6. 엔터티 정의

### 6.1 Book

책의 기본 단위입니다.

#### 역할
- 외부 공개 DB에서 조회한 철학 도서의 식별 단위
- 추천 결과의 중심 객체

#### 핵심 필드
- `bookId`
- `externalSource`
- `externalId`
- `title`
- `author`
- `publisher`
- `pubYear`
- `isbn`
- `language`
- `descriptionRaw`
- `categoryRaw`
- `sourceLink`

#### 설명
- `descriptionRaw`: 공개 DB의 도서 소개문
- `categoryRaw`: 외부 DB의 카테고리/주제 정보
- 책 본문 자체를 이해한 결과가 아니라 외부 메타데이터 기반 객체로 취급한다

---

### 6.2 Theme

사용자가 직접 보게 되는 추천 입구입니다.

#### 역할
- 첫 화면에서 선택하는 추천 주제
- 자유 입력을 연결하는 1차 목적지
- 선택 KPI 측정의 중심 단위

#### 핵심 필드
- `themeId`
- `name`
- `shortLabel`
- `group`
- `type`
- `description`
- `relatedConcepts`
- `exampleQueries`
- `priorityOrder`
- `isActive`

#### group
- 생활형
- 관심사형
- 사회·기술형
- 입문·출발형

#### type
- 감정형
- 상황형
- 질문형
- 주제형
- 책/저자 출발형

---

### 6.3 ConceptTag

자유 입력을 Theme에 연결하기 위한 의미 태그 레이어입니다.

#### 역할
- 사용자 입력의 의미를 느슨하게 구조화
- Theme 라우팅을 보조
- 정교한 철학 학술 분류체계가 아니라 추천용 의미 태그로 사용

#### 핵심 필드
- `conceptId`
- `name`
- `parentConcept`
- `aliases`

#### 예시
- 불안
- 실존
- 의미
- 사랑
- 죽음
- 자유
- 선택
- 정의
- 공정
- 행복
- 타자
- 정치
- 국가
- 윤리
- 기술
- AI
- 책임
- 인간
- 의식
- 자유의지

#### 원칙
- ConceptTag는 라우팅용 의미 태그로 사용한다
- 너무 무거운 철학 분류체계로 확장하지 않는다

---

### 6.4 CuratorPersona

추천 설명의 판단 기준과 말하는 방식을 고정하는 객체입니다.

#### 역할
- 결과 문체 일관성 유지
- 해석 범위 제한
- 신뢰 가능한 설명 톤 유지

#### 핵심 필드
- `personaId`
- `name`
- `summary`
- `voiceStyle`
- `reasoningRules`
- `forbiddenRules`

#### voiceStyle
- 짧고 명확함
- 철학적 맥락을 먼저 설명함
- 입문 난이도를 함께 설명함
- 서점 손님에게 직접 설명하듯 말함
- 과장하지 않음

---

### 6.5 InteractionLog

MVP 성과를 검증하기 위한 로그 구조입니다.

#### 역할
- 주제 선택 추적
- 추천 결과 상세 보기 추적
- 재방문 추적
- 추천 품질 개선 기초 데이터 확보

#### 핵심 필드
- `eventId`
- `sessionId`
- `userId`
- `eventType`
- `entryType`
- `themeId`
- `resolvedThemeIds`
- `inputQuery`
- `recommendedBookIds`
- `selectedBookId`
- `deviceType`
- `isReturnVisit`
- `createdAt`

---

### 6.6 CurationOutput

사용자에게 노출되는 설명형 추천 결과입니다.

#### 역할
- 책별 + 주제별 커스텀 설명 생성
- 추천을 설명 가능한 결과로 변환

#### 핵심 필드
- `bookId`
- `themeId`
- `inputQuery`
- `oneLineSummary`
- `philosophicalContext`
- `entryDifficulty`
- `whyThisBookForThisQuery`
- `limitedComparisonNote`
- `reasonTags`
- `generatedAt`

#### 출력 필드
- 제목
- 저자
- 한 줄 설명
- 철학적 맥락
- 입문 난이도
- 이번 질문에 맞는 이유

## 7. Theme 체계

초기 주제는 20개로 운영합니다.  
초기에는 모두 첫 화면에 노출하고, 주제 선택 KPI를 누적 관찰합니다.

### 7.1 생활형
1. 불안할 때 읽는 철학
2. 삶의 방향을 잃었을 때 읽는 철학
3. 관계가 힘들 때 읽는 철학
4. 사랑을 다시 생각하게 하는 철학
5. 죽음과 상실을 생각할 때 읽는 철학
6. 행복하게 산다는 게 무엇인지 묻는 철학
7. 내 선택이 맞는지 흔들릴 때 읽는 철학
8. 혼자 있는 시간이 버거울 때 읽는 철학

### 7.2 관심사형
9. AI와 인간을 생각하게 하는 철학
10. 기술과 윤리를 함께 생각하게 하는 철학
11. 자유의지를 생각하게 하는 철학
12. 인간이란 무엇인지 묻게 하는 철학
13. 의식과 마음을 생각하게 하는 철학

### 7.3 사회·기술형
14. 정의와 공정함을 생각하게 하는 철학
15. 정치와 사회를 다르게 보게 하는 철학
16. 책임과 판단을 생각하게 하는 철학
17. AI·자율주행 같은 기술 문제를 철학으로 보는 책

### 7.4 입문·출발형
18. 철학을 처음 시작하고 싶을 때
19. 한 권의 책에서 더 넓게 읽고 싶을 때
20. 한 철학자에서 시작하고 싶을 때

## 8. 관계 구조

### 8.1 Theme ↔ ConceptTag
- Theme는 하나 이상의 ConceptTag와 연결된다
- 자유 입력을 Theme로 라우팅할 때 사용한다

### 8.2 UserInput → ConceptTag → Theme
- 자유 입력에서 의미를 추출한다
- 가장 가까운 Theme 2~3개를 계산한다
- 바로 책을 추천하지 않고 먼저 Theme 후보를 제시한다

### 8.3 Theme → Book
- Theme에 맞는 Book 후보를 추출한다
- 이 관계는 고정 매핑보다 실시간 해석 기반이 적합하다

### 8.4 CuratorPersona → CurationOutput
- 모든 설명 결과는 추천자 페르소나의 기준을 따른다

### 8.5 CurationOutput → InteractionLog
- 어떤 결과가 생성되었고 사용자가 무엇을 눌렀는지 기록한다

---

## 9. 추천 처리 흐름

### 9.1 주제 선택형 흐름
1. 사용자가 주제 선택
2. Theme에 연결된 ConceptTag 확인
3. 외부 공개 메타데이터 기반으로 책 후보 탐색
4. CuratorPersona 기준으로 설명 생성
5. 추천 결과 출력

### 9.2 자유 입력형 흐름
1. 사용자가 질문 입력
2. AI가 입력에서 ConceptTag 추출
3. 가까운 Theme 2~3개 계산
4. Theme 후보를 먼저 제시
5. 사용자가 선택하거나 상위 Theme로 진행
6. 책 후보 탐색
7. CuratorPersona 기준으로 설명 생성
8. 결과 출력

### 9.3 해석 실패 시 처리
- 바로 책을 추천하지 않는다
- 가장 가까운 Theme 2~3개를 먼저 제안한다

## 12. 주제 운영 원칙

### 12.1 초기 운영 방식
- 주제 20개를 모두 첫 화면에 노출한다
- 더보기는 두지 않는다
- 초반에는 버릴지 말지 바로 판단하지 않는다

### 12.2 운영 관찰 지표
- 주제 선택 수
- 주제 선택 후 추천 상세 보기
- 주제 선택 후 재방문
- 자유 입력에서 특정 Theme로 자주 매핑되는 빈도

### 12.3 조정 원칙
- 안 눌린다고 바로 제거하지 않는다
- 일정 기간 데이터가 누적된 뒤 조정한다
- 클릭이 적더라도 특정 사용자층에 강하게 작동하는 주제는 보존 가능하다

## 13. 추천 화면 설계 원칙

### 13.1 디자인 목표
- 라이트함
- 깔끔함
- 높은 가독성
- 설명의 신뢰감
- 과장되지 않은 전문성

### 13.2 핵심 원칙
- 정보량은 충분하되 한 번에 과하게 몰아넣지 않는다
- 카드형보다 가벼운 타일/텍스트 중심 구조를 우선한다
- 제목, 저자, 설명, 추천 이유를 시각적으로 분리한다
- 강조는 최소화하고 위계만 분명히 둔다
- 여백을 충분히 확보한다
- 색은 절제하고 텍스트 중심으로 구성한다

### 13.3 추천 결과 카드 구성
- 제목
- 저자
- 한 줄 설명
- 철학적 맥락
- 입문 난이도
- 이번 질문에 맞는 이유
- 근거 태그 2~3개
- 상세 보기 버튼

## 14. 주제 선택 UX 원칙

### 14.1 핵심 원칙
- 주제는 큰 카드보다 가벼운 타일형으로 보여준다
- 20개를 한 덩어리로 나열하지 않고 그룹별로 묶는다
- 제목은 짧고 명확하게 유지한다
- 개수가 많아도 정돈돼 보이게 한다

### 14.2 노출 구조
1. 질문 유도 문장
2. 자유 입력창
3. 그룹별 주제 타일
   - 생활형
   - 관심사형
   - 사회·기술형
   - 입문·출발형

### 14.3 시각적 톤
- 밝은 배경
- 진한 회색 또는 검정 텍스트
- 포인트 색은 1개만 사용
- 과도한 아이콘, 그림, 배지 사용 금지
- 정렬과 여백을 우선한다

## 15. 반응형 설계 원칙

이 서비스는 핸드폰, 태블릿, PC를 모두 지원하는 반응형 구조를 기본으로 한다.

### 공통 원칙
- 모바일 우선으로 설계한다
- 정보 구조는 디바이스마다 바꾸지 않고 유지한다
- 화면 크기에 따라 그리드 열 수와 여백만 조정한다
- 주제 선택 UI는 텍스트 중심의 가벼운 타일형으로 구성한다

### 핸드폰
- 자유 입력창 상단
- 그룹별 제목 아래 주제 타일
- 1~2열 기본

### 태블릿
- 2열 중심
- 모바일보다 여백을 넓게

### PC
- 중앙 정렬된 컨테이너 사용
- 3~4열 기본
- 최대 너비 제한

## 16. 최종 정의

이 MVP는 다음과 같이 정의한다.

> 한국어 철학 도서를 공개 메타데이터 기반으로 탐색하고,  
> 사용자의 질문을 주제와 의미 태그로 해석한 뒤,  
> 철학 교수이자 서점 운영자인 추천자 페르소나의 관점으로  
> 설명형 추천 결과를 생성하는 서비스.
